# Phase 4 Storyboard — Design Review

**Reviewer:** advisor pre-close-out + manual heuristic pass.
**Date:** 2026-04-23
**Baseline:** 8-point grid, ≥4.5:1 contrast, focus states,
empty/loading/error states, keyboard nav, touch targets ≥44×44.

## Critical findings — addressed inline

1. **Initial-load PUT echo on the order endpoint.** First integration
   pass fired a `PUT /order` immediately after server data hydrated
   `order` from `null` → `string[]`. Fixed by reshaping
   `useDebouncedSave` to accept `T | null` and a `skipNextNonNull`
   ref, plus passing raw `order` (not `order ?? []`) from
   `StoryboardScreen`. Pinned by integration test
   "no PUT yet" + a dedicated `<StrictMode>` regression test.

2. **`useDebouncedSave` under React `<StrictMode>`.** Advisor flagged
   that the `useRef(true)` skip flag could be consumed by StrictMode's
   intentional double-effect on initial mount, then a real save would
   fire on the second pass with the same seeded array. Verified by a
   regression test that wraps the rendered tree in `<StrictMode>` and
   asserts `putOrderCalls.length === 0` after 500 ms. Test passed —
   safe in practice because the seed effect lands on a *subsequent*
   render (after server data resolves), not the initial mount cycle
   that StrictMode double-runs. Documented in `docs/design.md` so
   future copy-pasters know the contract.

3. **Playwright keyboard drag did not engage dnd-kit.** Switched to
   the mouse API with an explicit initial 4 px move that satisfies the
   `PointerSensor({ activationConstraint: { distance: 4 } })`
   threshold. The original `Space → Arrow → Space` keyboard flow stays
   wired up for accessibility but is exercised only by axe / a11y
   tests, not the headless E2E.

## Checklist pass

- **Contrast:** shadcn zinc tokens (Prepare baseline). Position badge
  uses `bg-primary text-primary-foreground` for ≥4.5:1.
- **Focus rings:** GripVertical drag handle is a `<button>` so it
  inherits shadcn's ring; `aria-label` reads
  "Drag frame N (filename). Press space to pick up, arrow keys to
  move, space to drop." for SR users.
- **Empty / loading / error:** loading and error reuse
  `JobProgressCard` (with `role="status"` / `role="alert"` per
  contract); empty state can't occur post-extend because `kling_test`
  is generated by the same job that gates the screen.
- **Keyboard:** dnd-kit `KeyboardSensor` + `sortableKeyboardCoordinates`
  registered. Tab → drag handle, Space pick up, Arrow move, Space drop.
- **Touch targets:** drag handle is 44×44 (`h-11 w-11` on the button
  wrapper).
- **8-point grid:** `gap-4`, `p-4`, `mb-3` throughout the grid.
- **SR live region:** "Saving… / Order saved" status uses
  `role="status" aria-live="polite"` so the debounced save state is
  announced.

## Non-critical (flagged for follow-up sub-plans / Phase 6)

1. **`useDebouncedSave` swallows errors silently.** A 5xx on
   `PUT /order` will flip `isPending` back to false without any user
   feedback. Phase 6 polish: surface a toast or revert the local state
   to the server's last-known order. Not blocking — worst case the
   user re-drags one frame and the next PUT recovers.
2. **500 ms `waitForTimeout` brittleness in the Playwright spec.** The
   debounce is 300 ms and the PUT is mocked, so 500 ms covers the
   round trip on most machines. If CI starts to flake here, switch to
   `waitForRequest('**/order', { method: 'PUT' })` instead of a fixed
   sleep.
3. **`order.json` orphan if a frame name reference drifts.** If the
   Generate sub-plan ever renames `1.jpg` → `01.jpg` mid-flight, the
   Storyboard order will silently fall back to numeric sort (because
   `_ordered_frames` filters to existing files). Not currently
   reachable — Prepare's output names are stable — but Generate's
   sub-plan must re-acknowledge this contract.
4. **dnd-kit bundle weight.** ~30 KB gzipped for the trio. Acceptable
   for now (only Storyboard uses it). If Review's re-order surface
   doesn't ship in the same wizard, consider lazy-loading the
   Storyboard route to keep the initial bundle lean. Phase 6.
